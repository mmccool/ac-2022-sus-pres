Smart Building Use Cases

Smart Buildings can:

  • Reduce energy consumption when rooms are unoccupied
  • Anticipate and learn energy usage patterns
  • Shift energy demand to shave peak consumption
  • Manage power self-generation and grid interfacing
Web of Things has been used in Building Information Management systems from Siemens AG and Takanaka Corporation.

Smart Building Observations

  • Using the cloud for local energy-saving automations MAY be a net loss!
  • Standards are needed to support secure local communication.
  • Standards and business models need to be aligned to encourage adoption.
  • HOWEVER:
    • Local is not always better
    • Sometimes cloud systems can be better managed and more energy-efficient
    • What is really needed are metrics to allow good energy-usage decisions

Cloud/Edge Observations

  • Communication costs energy
  • Cloud has higher latency than local compute
  • Latency is serial overhead
  • Serial overhead limits parallel speedup
  • Local/distributed (e.g. edge) computing needs a more flexible security/trust model

Discussion

  • "Localizing" compute when possible may improve sustainability
    • Need metrics to make informed decisions
    • Need local security PKI infrastructure
    • Extended trust model supporting other computing resources
  • Sustainability considerations and wide review in process?
  • Business models and incentives need to be considered as well.
    • A sustainable standard that is not adopted is not useful
